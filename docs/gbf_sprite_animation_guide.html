What GBFAP Is and How It Actually Works
📦 The Project

GBFAP (GBF Animation Player) by MizaGBF is an open-source MIT-licensed tool that reverse-engineered Cygames' animation system and can play every character's battle animation in-browser. It's what you saw at mizagbf.github.io/GBFAP.

The project is not standalone — it depends on three sibling repos: GBFML (markup/layout), GBFAL (asset lookup), and GBFCP (CORS proxy server). The animation player itself is the core, but it needs the proxy to actually fetch GBF's assets.

MIT LicenseOpen SourceSelf-hostable

🔑 The Core Problem: CORS

GBF assets live on Cygames' CDN (game-a1.granbluefantasy.jp) and have Cross-Origin policies that block direct browser fetches from other domains.

GBFAP solves this with GBFCP — a Python proxy server that sits between the browser and GBF's CDN, stripping the CORS restriction. On the public GitHub Pages version, Miza runs this proxy on a small server. When you self-host, you run your own proxy locally.

⚠️ You cannot directly load game-a.granbluefantasy.jp assets in your game without either running your own CORS proxy or downloading the assets to your own server first.
⚙️ The Animation Engine — CreateJS

GBF's battle animations are CreateJS MovieClips — specifically createjs.MovieClip from the EaselJS library (part of the CreateJS suite originally built for Flash/HTML5 Canvas output from Adobe Animate). The asset file for each character (unit_[ID].js) is a self-contained JS class that:

PART 1 — Spritesheet definition

Defines which rectangular region of the PNG corresponds to each "part" (head, torso, left arm, weapon, effect layer 1, etc.). Each frame is defined as [x, y, width, height, imageIndex, regX, regY].

PART 2 — Animation clips

Named clips like atk, sp, damage, die, idle define which frame sequences to play for each action. Clips can loop or play once.

PART 3 — MovieClip tweens

A createjs.MovieClip wraps the spritesheet animation and adds tween keyframes — position, scale, rotation, alpha per body part, per frame — via createjs.Tween.

GBF Asset URL Patterns
🗂️ Character ID Format

GBF character IDs are 10-digit numbers. The first digit encodes rarity/type, the next 3 are the character family, the next 4 are the variation/skin, and the last 2 are usually 00. Examples: 3040028000 (SSR Zeta), 3030003000 (SR Rosetta).

SSR = 304...  ·  SR = 303...  ·  R = 302...  ·  Main Character = 399...

📂 Asset URL Patterns (via CORS proxy)

All URLs below are on game-a[1–4].granbluefantasy.jp. In GBFAP, they're prefixed with the CORS proxy URL.

Battle animation JS file (the CreateJS MovieClip definition):

https://game-a.granbluefantasy.jp/assets_en/js/nsp_js/unit_[CHARACTER_ID].js
e.g. unit_3040028000.js — contains the full CreateJS animation definition

Battle spritesheet PNG (the texture atlas for the animation):

https://game-a.granbluefantasy.jp/assets_en/img/sp/assets/npc/sd/[CHARACTER_ID].png
e.g. 3040028000.png — all animation frames packed into one image

Attack effect spritesheet (separate from character sprite):

https://game-a.granbluefantasy.jp/assets_en/img/sp/assets/npc/sd/[CHARACTER_ID]_atk.png
Attack VFX sheet — slash trails, hit sparks, element bursts

Skill (SP) effect spritesheet:

https://game-a.granbluefantasy.jp/assets_en/img/sp/assets/npc/sd/[CHARACTER_ID]_sp.png
Skill VFX — element-flavored auras, projectiles, impact frames

Charge Attack (Ougi) spritesheet:

https://game-a.granbluefantasy.jp/assets_en/img/sp/assets/npc/sd/[CHARACTER_ID]_soul.png

Portrait (story/scene art):

https://game-a.granbluefantasy.jp/assets_en/img_mid/sp/quest/scene/character/body/[CHARACTER_ID]_b.png

https://game-a.granbluefantasy.jp/assets_en/img/sp/ui/thumb/[CHARACTER_ID].jpg
How to Use GBFAP in Your Own Game
🗺️ The Two Paths

Path A — Embed GBFAP's player directly

GBFAP provides a demo.html that shows the minimum code to embed the animation player as an iframe or component into another page. load_player() is the entry function. This is the fastest path if you just want the player to appear in your game UI.

✓ Works immediately  ·  ✓ Gets all updates automatically  ·  ✗ Depends on Miza's proxy (limited bandwidth)  ·  ✗ Can't customize deeply

Path B — Self-host the full stack

Clone GBFAP + GBFML + GBFAL, run GBFCP locally (Python proxy), configure config.json to point to your proxy, serve with any HTTP server. Full control. Can download assets locally.

✓ Full control  ·  ✓ No bandwidth limits  ·  ✓ Can cache assets locally  ·  ✗ Needs server setup  ·  ✗ Legal grey area with Cygames assets
📋 Path A: Embed the Player — Step by Step
  1. Include GBFAP dependencies in your page's <head>: 
    <!-- CreateJS (required by GBF's animation files) -->
<script src="https://code.createjs.com/1.0.0/createjs.min.js"></script>
<!-- Backbone + Underscore (also required by GBF's animation system) -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/backbone.js/1.4.0/backbone-min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/underscore.js/1.13.1/underscore-min.js"></script>
<!-- GBFML utility (required by GBFAP's player) -->
<script src="https://mizagbf.github.io/GBFML/js/util.js"></script>
<!-- GBFAP player script -->
<script src="https://mizagbf.github.io/GBFAP/js/player.js"></script>
  2. Add a canvas container in your HTML: 
    <div id="gbfap-container" style="width:480px;height:270px">
  <canvas id="gbfap-canvas"></canvas>
</div>
  3. Initialize the player with a character ID and animation type: 
    // After page load:
load_player({
  use_game_config: "proxy",       // use Miza's CORS proxy
  canvas: "gbfap-canvas",
  id: "3040028000",               // character ID (Zeta in this case)
  type: "npc",                    // "npc" = character, "summon" = summon
  animation: "atk"                // "atk", "sp", "sp2", "damage", "die", "idle"
});
  4. To switch animations at runtime: 
    // Tell the player to play a different animation
player_switch_animation("sp");   // switch to skill animation
player_switch_animation("atk");  // switch back to attack
  5. To load a different character: 
    // Reload the player with a new character ID
load_player({ id: "3040001000", type: "npc", animation: "atk" });
⚠️ Miza's proxy has limited bandwidth and asks users not to overload it. For production use, run your own GBFCP proxy. Also: using Cygames' assets in your own game without permission is a legal grey area — treat it like fan content, not commercial use.
📋 Path B: Self-Host — Setup Steps
  1. Clone the required repos into a common root folder: 
    mkdir gbf-project && cd gbf-project
git clone https://github.com/MizaGBF/GBFAP
git clone https://github.com/MizaGBF/GBFML
git clone https://github.com/MizaGBF/GBFCP   # CORS proxy
# optional but useful:
git clone https://github.com/MizaGBF/GBFAL
  2. Install and start the CORS proxy (Python 3.13 required): 
    cd GBFCP
pip install -r requirements.txt
python app.py           # starts proxy on port 8001
# or for debug mode:
python app.py -debug
  3. Edit GBFAP/config.json — change use_game_config to "local" and set corsProxy to "http://localhost:8001/": 
    {
  "use_game_config": "local",
  "game_config": {
    "local": {
      "xjsUri": "CORS/https://game-a.granbluefantasy.jp/...",
      "jsUri":  "CORS/https://game-a.granbluefantasy.jp/...",
      "imgUri": "CORS/https://game-a.granbluefantasy.jp/...",
      "corsProxy": "http://localhost:8001/"
    }
  }
}
  4. Start a local HTTP server from the root folder (NOT inside GBFAP): 
    cd ..   # back to gbf-project root
python -m http.server 8080
# then open: http://localhost:8080/GBFAP/
  5. Optional: Download assets locally using the updater tool to avoid live CDN fetches: 
    cd GBFAP
python updater.py -r          # scan for new content
python updater.py -d          # download all assets (time-consuming!)
python updater.py -d 3040028000  # or download one specific character

💡 In GBFAP, you can see all available animation clips for a character using the playlist button (P). Some characters reuse another character's animations when theirs don't exist — GBFAP does its best to match them automatically.
🔄 Attack Multiplicity (DA / TA)

GBF doesn't have separate "DA animation" and "TA animation" files. DA and TA are handled by the game's battle logic, not the animation system:

  • Normal attack: Play atk animation once
  • Double Attack (DA): Play atk animation, then after it completes, trigger the attack effect sheet a second time at a slightly different position
  • Triple Attack (TA): Three sequential atk triggers with staggered timing (~150ms apart)
  • Full Chain Burst: All party members TA simultaneously → combined Charge Attack fires

The visual "second hit" for DA is the attack effect animation ([ID]_atk.png) played again — not a new character animation. The character sprite itself only lunges once; the hit VFX is what repeats.

The GBFAP player renders onto a Canvas element sitting in your battle stage. Your game logic sends animation commands to it. On top of the canvas, your CSS/JS VFX layer handles hit flashes, screen effects, damage numbers — things not in the character animation itself.

✅ What GBFAP Provides
  • Character idle loop animations
  • Attack animations (lunge + weapon slash)
  • Skill casting animations (element-specific)
  • Charge Attack / Ougi cinematics
  • Damage / knocked-out animations
  • Enemy animations
  • Summon call animations
  • Weapon animations (separate asset category)
  • All as real GBF art — identical to the game

💻 Minimal Integration Code

This is the cleanest pattern for triggering GBFAP from your battle system:

// ── 1. Setup: map your characters to their GBF IDs ──────────────────────
const PARTY = [
  { name: 'Zeta',    id: '3040028000', element: 'fire'  },
  { name: 'Narmaya', id: '3040213000', element: 'dark'  },
  { name: 'Yodarha', id: '3040110000', element: 'water' },
];

// ── 2. Initialize GBFAP for each character slot ─────────────────────────
// Each slot gets its own canvas element
PARTY.forEach((char, slotIdx) => {
  load_player({
    use_game_config: "proxy",     // or "local" if self-hosting
    canvas: `char-canvas-${slotIdx}`,
    id: char.id,
    type: "npc",
    animation: "idle"             // start in idle loop
  });
});

// ── 3. Trigger animations from your battle system ───────────────────────
function onAttack(slotIdx, attackType) {
  const canvas = `char-canvas-${slotIdx}`;
  
  // Switch to attack animation, then return to idle when done
  player_switch_animation("atk", canvas);
  
  // DA: trigger VFX second hit after ~400ms (atk anim duration)
  if (attackType === 'da') {
    setTimeout(() => triggerHitVFX(slotIdx, 2), 400);
  }
  
  // TA: trigger 3 hit VFX in sequence
  if (attackType === 'ta') {
    [0, 150, 300].forEach((delay, i) => {
      setTimeout(() => triggerHitVFX(slotIdx, i + 1), delay + 350);
    });
  }
  
  // Return to idle after attack completes (~800ms)
  setTimeout(() => player_switch_animation("idle", canvas), 800);
}

function onSkill(slotIdx, skillNum) {
  const canvas = `char-canvas-${slotIdx}`;
  const animName = skillNum === 1 ? "sp" : "sp2";
  player_switch_animation(animName, canvas);
  // Skills typically run longer — return to idle after ~1500ms
  setTimeout(() => player_switch_animation("idle", canvas), 1500);
}

function onChargeAttack(slotIdx) {
  const canvas = `char-canvas-${slotIdx}`;
  player_switch_animation("mortal", canvas);  // or "ougi"
  // CA animations vary in length — listen for animation end if possible
  setTimeout(() => player_switch_animation("idle", canvas), 3500);
}

function onDamaged(slotIdx) {
  const canvas = `char-canvas-${slotIdx}`;
  player_switch_animation("damage", canvas);
  setTimeout(() => player_switch_animation("idle", canvas), 500);
}

Bottom line for your game: If it's a fan project — embed GBFAP with load_player() and map your skill system to animation names. If you want full control without CDN dependency, self-host with your own GBFCP proxy and optionally download assets locally with the updater tool. The battle VFX layer (hit flashes, damage numbers, screen effects) you already have from the previous preview — GBFAP handles the character animation part on top of that.
